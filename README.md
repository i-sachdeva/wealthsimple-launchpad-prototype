# Explainable Phishing Detection Prototype

This is a lightweight concept prototype demonstrating how phishing detection systems can be made more transparent and trustworthy.

## Idea
Most email platforms flag suspicious messages but don’t clearly explain *why*.  
This prototype shows how backend signals can be translated into simple, user-facing reasoning.

## Features
- Risk scoring based on:
  - Domain similarity
  - Suspicious links
  - Urgency language
  - Credential requests
- Clear explanation of why an email is flagged
- Recommended action based on risk level

## Purpose
This is not a production security system.  
It is a product prototype designed to demonstrate:
- Explainable security
- Backend decision logic
- User trust in high-stakes systems

## How to use
Open the HTML file and click “Analyze email” or load an example.

## Context
Built as part of an application exploring how to design reliable backend systems that handle sensitive data.
